# DreamCore プロダクトビジョン

> **「夢をゲームにする」**

このドキュメントは、DreamCore の思想、目標、将来像をまとめたものです。
新しくチームに参加するエンジニアや、意思決定に迷ったときの指針として活用してください。

---

## ミッション

### ゲーム開発の民主化

DreamCore は、**誰もがゲームを作れる世界**を目指しています。

- プログラミング経験ゼロの人でも
- 高性能なPCを持っていない人でも
- 開発環境を整えられない地域の人でも

**「話すだけ」でゲームが作れる**体験を提供します。

### クリエイターの支援

私たちは単なるツール提供者ではなく、**クリエイターの味方**です。

- 技術的なハードルを取り除く
- 収益化の機会を提供する
- コミュニティで成長を支援する

## 創業の原点

### UGC ゲームプラットフォームへの可能性

DreamCore の創業者は、以前フォートナイトや Roblox などの UGC ゲームプラットフォーム向けにゲームを作る Studio を経営していた。そこで、ゲームプラットフォームの可能性を強く感じていた。

そこに AI が登場。**AI ネイティブなプラットフォーム**が作れると確信。「ゲーム版 TikTok」のような存在を作りたいと考えていた。

### 「ゲームのプリクラ化」への気づき

元スクウェア・エニックス社長の和田洋一氏との雑談がきっかけで、コアバリューが明確になった。

DreamCore を見せたり、いろんなゲームやテクノロジーの話をしていたとき、和田氏が言った：

> 「TikTok はプリクラなんだ」

プリクラが写真を「撮られるもの」から「コミュニケーションツール」に変えた。TikTok も動画をそうした。

その話を聞いて**ハッとした**。DreamCore は「ゲーム版プリクラ」になれる。**ゲームを作ること自体がコミュニケーションになる**。これがコアバリューなんだと気づいた。

この気づきは、腹落ちしつつも時間が経つほどに**じわじわと深まっていった**。

## コアバリュー

### 1. シンプルでも完成していること

DreamCore で作られるゲームは：

- **30秒で遊べる** - 複雑なチュートリアル不要
- **ルールがわかりやすい** - 直感的に遊び方がわかる
- **ユニークなアイデア** - 個性が光る作品

「大作」よりも「完成した小さなゲーム」を重視します。

### 2. コミュニティ文化

DreamCore のコミュニティは：

- **コラボ・リミックス推奨** - 他の作品から学び、発展させる
- **建設的フィードバック** - 批判ではなく提案で成長を支援
- **楽しさ優先** - 競争よりも楽しむことを重視

### 3. 創造性の解放

私たちが絶対に避けること：

- テンプレートばかりで個性が出ない
- 一部の人だけが得をする仕組み
- 使いにくい・不安定なツール

### 「わかってる」クリエイターの特徴

DreamCore で活躍するクリエイターに共通する特徴：

| 特徴 | 説明 |
|------|------|
| **誰かのために作る** | 特定の誰か、仲間、まだ見ぬ誰か。顔が浮かぶ相手に向けて作っている |
| **自分を出す** | 作品に「自分らしさ」が滲み出ている |
| **完成させる** | 途中で諦めず、小さくても完成まで持っていく |
| **楽しんでいる** | 作ること自体を楽しんでいる |
| **短く作る** | シンプルで遊びやすいゲームを作る |

## ゲームの「プリクラ化」

### 概念

プリクラは、写真を「撮られるもの」から「コミュニケーションツール」に変えました。
DreamCore は、ゲームを「遊ぶもの」から**「作るもの」**に変えます。

**ゲームを作ること自体がコミュニケーションになる世界**を目指しています。

### 変化の本質

| キーワード | 意味 |
|-----------|------|
| **表現の民主化** | 誰でも作り手になれる |
| **気軽さ** | 完璧じゃなくていい、楽しければいい |
| **つながり** | 作ることで人と繋がれる |
| **自己開示** | 自分を表現し、理解してもらえる |

### 「盛れる」とは「自己のIP化」

プリクラや TikTok が行ってきたのは、**自己のIP化・キャラクター化**。
DreamCore でも同じことを実現したい。

- **ペルソナの確立** - 「こういうゲームを作る人」として認識される
- **作品が自分の分身** - ゲームが「自分らしさ」を伝える媒体になる
- **ファンがつく** - 自分の作品を待ってくれる人ができる
- **収益化** - 自分のIPからお金が生まれる

**自分自身をゲームのキャラクターにしてしまう。** 実写以外の方法で、自己表現が手軽に可能になる。

### Remix という名の会話

プリクラは「一緒に撮る」から楽しい。DreamCore では：

- **Remix で会話** - 誰かの作品に返歌のように返す
- **遊んでもらう** - 自分が作ったものを誰かが遊んでくれる喜び
- **反応をもらう** - いいねやコメントで繋がる
- **お題で競う** - 同じテーマでそれぞれが作って見せ合う

リアルタイムで一緒に作るわけじゃないけど、**作品を通じた非同期のコミュニケーション**が生まれる。

### 原体験：息子とのゲーム作り

創業者が「ゲームを作ることがコミュニケーションになる」を確信した体験：

小学生の息子のアイデアでアクションゲームを作った。息子は、**目の前で自分のアイデアがゲームになっていく**のを見て、すごく喜んでくれた。

そのとき感じたこと：
- **嬉しかった** - 純粋に嬉しかった
- **確信した** - DreamCore の価値、ゲームの力、子供だってクリエイターになれる可能性、そして自分が進むべき道
- **驚いた** - 子供の反応の大きさ、こんなに簡単に作れたこと、ゲームをきっかけに会話が生まれたこと、自分自身がこんなに嬉しいとは思わなかったこと

### TikTok との決定的な違い

**インタラクティブ性**。

TikTok は見る。プリクラは見る。でもゲームは**遊ぶ**。

- 受け手が能動的に体験する
- ルールがあって、勝ち負けや目的がある
- 作品が残り、継続的に遊ばれる

ただ消費されるコンテンツじゃなく、**体験として残る**。

### ゲームが「新しい言語」になる未来

「ゲームを作ることがコミュニケーション」が当たり前になった世界では：

- **新しい言語** - 文字や動画のように、ゲームが表現手段のひとつになる
- **記憶の保存** - 写真やアルバムのように、思い出をゲームで残す
- **商業を超える** - 売るためじゃない、伝えるためのゲーム
- **クリエイターの爆発** - 誰もがクリエイターである世界

## 技術哲学

### スピードと安定性の両立

| 優先度 | 項目 | 理由 |
|--------|------|------|
| 1 | **スピード** | 生成・レスポンスの速さがUXの核心 |
| 2 | **安定性** | クラッシュしない信頼性 |
| 3 | **スケーラビリティ** | 大量ユーザーに対応できる設計 |

### AI の使い方

- **ユーザーの意図を汲み取る** - 曖昧な指示でも理解してくれる
- **適当でもOK** - 完璧な指示がなくても動く

AI は「完璧な指示を要求するツール」ではなく、「一緒に考えてくれるパートナー」です。

## 開発原則

### ユーザーファースト

技術的な美しさよりも、ユーザーにとっての価値を優先します。

```
❌ 「この実装はエレガントだ」
✅ 「ユーザーはこれで幸せか？」
```

### 高速イテレーション

考えるより、まず作る。素早くリリースしてフィードバックを得る。

```
❌ 完璧になるまで待つ
✅ 動くものを出して改善する
```

### シンプルに保つ勇気

機能追加の誘惑に抗う。複雑さは敵。

```
❌ 「この機能もあった方がいい」
✅ 「なくても目的は達成できるか？」
```

### グローバルを意識

日本だけでなく、世界中のユーザーを想定する。

- 多言語対応（i18n）を標準で
- 文化的な前提を置かない
- 低スペック環境でも動くように

## チーム文化

### 価値観

- **スピードと実行力** - 考えるより先に手を動かす
- **ユーザーへの共感** - ユーザーの気持ちを理解する
- **オープンなコミュニケーション** - 透明性と議論を重視

### エンジニアへのメッセージ

1. **ユーザー体験を最優先に**
   - 技術的な美しさより、ユーザー価値

2. **シンプルに保つ勇気**
   - 複雑にしたくなる誘惑に抗う

3. **グローバルを意識**
   - 日本だけでなく世界を見る

4. **クリエイターの味方であれ**
   - ビジネスよりクリエイターの成功を

### クリエイターへのメッセージ

DreamCore を使うすべてのクリエイターに伝えたいこと：

> **楽しんで。完成させて。シェアして。自由に。**

難しく考えなくていい。完璧じゃなくていい。誰かに見せたくなったら、それが一番いいタイミング。

## 絶対に避けること（アンチパターン）

| アンチパターン | 理由 |
|---------------|------|
| **十分に使えない状態でのリリース** | 不安定・重い・複雑なツールは信頼を失う |
| **クリエイターへの不公平** | 一部の人だけが得をする仕組みはNG |
| **創造性の制限** | テンプレートばかりで個性が出ないのは本末転倒 |
| **有害コンテンツの温床** | 暴力・差別的ゲームが溜まる場所にしない |
