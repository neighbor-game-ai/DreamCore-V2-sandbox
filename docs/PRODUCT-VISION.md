# DreamCore プロダクトビジョン

> **「夢をゲームにする」**

このドキュメントは、DreamCore の思想、目標、将来像をまとめたものです。
新しくチームに参加するエンジニアや、意思決定に迷ったときの指針として活用してください。

---

## ミッション

### ゲーム開発の民主化

DreamCore は、**誰もがゲームを作れる世界**を目指しています。

- プログラミング経験ゼロの人でも
- 高性能なPCを持っていない人でも
- 開発環境を整えられない地域の人でも

**「話すだけ」でゲームが作れる**体験を提供します。

### クリエイターの支援

私たちは単なるツール提供者ではなく、**クリエイターの味方**です。

- 技術的なハードルを取り除く
- 収益化の機会を提供する
- コミュニティで成長を支援する

## コアバリュー

### 1. シンプルでも完成していること

DreamCore で作られるゲームは：

- **30秒で遊べる** - 複雑なチュートリアル不要
- **ルールがわかりやすい** - 直感的に遊び方がわかる
- **ユニークなアイデア** - 個性が光る作品

「大作」よりも「完成した小さなゲーム」を重視します。

### 2. コミュニティ文化

DreamCore のコミュニティは：

- **コラボ・リミックス推奨** - 他の作品から学び、発展させる
- **建設的フィードバック** - 批判ではなく提案で成長を支援
- **楽しさ優先** - 競争よりも楽しむことを重視

### 3. 創造性の解放

私たちが絶対に避けること：

- テンプレートばかりで個性が出ない
- 一部の人だけが得をする仕組み
- 使いにくい・不安定なツール

## 技術哲学

### スピードと安定性の両立

| 優先度 | 項目 | 理由 |
|--------|------|------|
| 1 | **スピード** | 生成・レスポンスの速さがUXの核心 |
| 2 | **安定性** | クラッシュしない信頼性 |
| 3 | **スケーラビリティ** | 大量ユーザーに対応できる設計 |

### AI の使い方

- **ユーザーの意図を汲み取る** - 曖昧な指示でも理解してくれる
- **適当でもOK** - 完璧な指示がなくても動く

AI は「完璧な指示を要求するツール」ではなく、「一緒に考えてくれるパートナー」です。

---

## 開発原則

### ユーザーファースト

技術的な美しさよりも、ユーザーにとっての価値を優先します。

```
❌ 「この実装はエレガントだ」
✅ 「ユーザーはこれで幸せか？」
```

### 高速イテレーション

考えるより、まず作る。素早くリリースしてフィードバックを得る。

```
❌ 完璧になるまで待つ
✅ 動くものを出して改善する
```

### シンプルに保つ勇気

機能追加の誘惑に抗う。複雑さは敵。

```
❌ 「この機能もあった方がいい」
✅ 「なくても目的は達成できるか？」
```

### グローバルを意識

日本だけでなく、世界中のユーザーを想定する。

- 多言語対応（i18n）を標準で
- 文化的な前提を置かない
- 低スペック環境でも動くように

## ビジネスモデル

### 方向性

| 対象 | モデル |
|------|--------|
| **プレイヤー** | 基本無料 + TikTok型広告 |
| **クリエイター** | Roblox型アイテム課金 |
| **将来** | アセット販売マーケットプレイス |

### 哲学

- プレイヤーは無料で遊べる
- クリエイターが収益を得られる仕組み
- プラットフォームとクリエイターの Win-Win

   - 言語や地域を超えて

## 絶対に避けること（アンチパターン）

| アンチパターン | 理由 |
|---------------|------|
| **十分に使えない状態でのリリース** | 不安定・重い・複雑なツールは信頼を失う |
| **クリエイターへの不公平** | 一部の人だけが得をする仕組みはNG |
| **創造性の制限** | テンプレートばかりで個性が出ないのは本末転倒 |
| **有害コンテンツの温床** | 暴力・差別的ゲームが溜まる場所にしない |
