# DreamCore 翻訳ガイド

## プロダクト概要

### DreamCore とは

**「おしゃべりするだけでゲームが作れる」AIゲーム制作サービス**

ユーザーがチャットで「こんなゲームを作りたい」と伝えると、AIが自動でブラウザゲームを生成する。プログラミング知識は一切不要。

### ビジョン

「ゲームを作りたい」という夢を持つすべての人に、その機会を届ける。

技術的なハードルをAIが取り除き、アイデアさえあれば誰でもゲームクリエイターになれる世界を目指す。

---

## 主な機能

| 機能 | 説明 | ユーザーの言葉 |
|------|------|----------------|
| **ゲーム作成** | チャットでAIに指示してゲームを作る | 「シューティングゲームを作って」 |
| **リアルタイムプレビュー** | 作成中のゲームをすぐに試せる | 「動かしてみる」 |
| **ゲーム公開** | 作ったゲームをURLで共有できる | 「友達に見せたい」 |
| **発見（Discover）** | 他の人が作ったゲームを遊べる | 「みんなのゲームを見る」 |
| **マイページ** | 自分のゲーム一覧とプロフィール | 「自分のページ」 |

## ターゲットユーザー

### メインターゲット

**ITリテラシーが低い一般ユーザー**

| 属性 | 詳細 |
|------|------|
| **年齢** | 10代〜40代（幅広い） |
| **ITスキル** | スマホでSNSを使う程度 |
| **プログラミング経験** | なし |
| **ゲームへの興味** | 遊ぶのは好き、作ってみたいと思ったことがある |

### ユーザーの特徴

- 「プログラミング」と聞くと難しそうと感じる
- 専門用語を見ると不安になる
- 直感的に操作できることを期待する
- 失敗を恐れる（「壊しちゃうかも」）

### ユーザーが求めるもの

- 簡単さ（「私でもできる」）
- 安心感（「失敗しても大丈夫」）
- 達成感（「ゲームが作れた！」）
- 承認（「すごいね」と言われたい）

## 翻訳の原則

### 1. やさしい言葉を使う

| ❌ 避ける | ✅ 使う |
|-----------|---------|
| デプロイ | 公開 |
| オーセンティケーション | ログイン |
| コンパイル | （使わない） |
| プラットフォーム | サービス / アプリ |
| インターフェース | 画面 |
| リポジトリ | （使わない） |
| API | （使わない） |
| サーバー | （使わない、または「システム」） |

### 2. 短く、具体的に

| ❌ 長い | ✅ 短い |
|---------|---------|
| 認証に失敗しました | ログインできませんでした |
| システムエラーが発生しました | うまくいきませんでした |
| 再試行してください | もう一度 |
| 処理を実行中です | 読み込み中... |

### 3. 親しみやすいトーン

| ❌ 堅い | ✅ やわらかい |
|---------|---------------|
| 本日の上限に達しました | 今日はここまで！ |
| 削除を確認 | このゲームを削除する？ |
| 後から変更可能です | あとから変えられます |
| エラーが発生しました | うまくいきませんでした |

### 4. 行動を促す言葉

| ❌ 説明的 | ✅ 行動的 |
|----------|----------|
| Googleでサインイン | Googleではじめる |
| メールを送信しました | メールを送りました！リンクをクリックしてください |
| ゲームを作成 | あたらしいゲームをつくる |

### 5. 安心感を与える

| シーン | 表現例 |
|--------|--------|
| 名前入力 | 「あとから変えられます」 |
| 削除確認 | 「削除すると元にもどせません」（脅しではなく事実として） |
| エラー時 | 「もう一度お試しください」（責めない） |

## 言語別の注意点

### 英語 (en)

- シンプルな単語を選ぶ（make > create > develop）
- 短い文を心がける
- "you" を積極的に使う（親しみやすさ）
- 能動態を使う（"Your game was created" → "You made a game!"）

**トーンの参考**: Duolingo, Canva（カジュアルで励ます感じ）

### 日本語 (ja)

- ひらがなを多めに（むずかしい → 難しい ではなく ひらがなで）
- 「〜しましょう」「〜できます」で励ます
- カタカナ語は最小限（ログイン、OK は許容）
- 敬語は「です・ます」止め（「でございます」は堅すぎ）

**トーンの参考**: LINE, Nintendo（親しみやすく丁寧）

### 中国語 (zh)

- 口語的な表現を使う（聊着天 > 通过聊天）
- 「啦」「吧」「呢」などの語気助詞で親しみやすく
- 簡体字を使用（中国大陸メイン）
- 「您」より「你」（フレンドリーに）

**トーンの参考**: 抖音, 小红书（若者向けカジュアル）

### スペイン語 (es)

- ラテンアメリカ向けのニュートラルなスペイン語
- 「tú」を使用（「usted」は堅すぎ）
- 感嘆符「¡」を適切に使う
- 親しみやすく、励ます表現

**トーンの参考**: Duolingo ラテンアメリカ版

### ポルトガル語 (pt)

- ブラジルポルトガル語（pt-BR）
- 「você」を使用
- 口語的で親しみやすい表現（「Não dá pra」など）
- 若者言葉を適度に取り入れる

**トーンの参考**: TikTok ブラジル、Nubank

### 韓国語 (ko)

- 해요体を使用（친근하고 부드럽게）
- 「~요」で終わる丁寧だけど親しみやすい表現
- 英語のカタカナ語は韓国語読みに（게임、로그인）
- 絵文字のようなニュアンスを言葉で表現

**トーンの参考**: 카카오, 네이버（韓国の大手サービス）

## キー命名規則

```
page.{pageName}.{element}  - ページ固有のテキスト
common.{element}           - 共通テキスト
nav.{item}                 - ナビゲーション
button.{action}            - ボタン
modal.{name}.{element}     - モーダル内
error.{type}               - エラーメッセージ
message.{type}             - 成功メッセージ等
placeholder.{field}        - 入力欄のプレースホルダー
filter.{type}              - フィルターオプション
stats.{type}               - 統計ラベル
status.{state}             - 接続状態など
```

## 翻訳の追加方法

1. `en.json` に英語を追加
2. `ja.json` に日本語を追加
3. `zh.json` に中国語を追加
4. このガイドの用語集に新しい用語があれば追加

**重要**: 機械翻訳をそのまま使わない。このガイドの原則に沿っているか必ず確認する。
