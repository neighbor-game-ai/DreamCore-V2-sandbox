# 作業記録スキル

## 概要

プロジェクトごとに作業履歴を記録・管理するスキルです。

## トリガー

以下のような依頼で実行:
- 「作業を記録して」
- 「履歴を更新して」
- 「やったことをメモして」
- 「ログを残して」

## ファイル構成

```
{プロジェクトルート}/
├── .claude/
│   ├── logs/          ← 詳細な作業ログ（日付別）
│   └── plans/         ← 計画ファイル
└── TODO.md            ← 概要 + 参照リンク
```

## 実行手順

### 1. ディレクトリ確認・作成

```bash
mkdir -p .claude/logs .claude/plans
```

### 2. 詳細ログを作成

ファイル名: `.claude/logs/YYYY-MM-DD-タスク名.md`

```markdown
# タスク名

**日付:** YYYY-MM-DD
**作業者:** Claude

## 実施内容

- 具体的な作業内容
- 変更した機能や修正

## 発見した問題と対応

- 問題1: 内容 → 対応方法
- 問題2: 内容 → 対応方法

## 変更ファイル一覧

| ファイル | 変更内容 |
|---------|---------|
| path/to/file.ts | 機能追加 |
| path/to/other.ts | バグ修正 |

## 学び・注意点

- 今後気をつけること
- 発見した知見
```

### 3. TODO.md を更新

プロジェクトルートの `TODO.md` に概要を追記:

```markdown
## YYYY-MM-DD: 作業タイトル

**詳細:** `.claude/logs/YYYY-MM-DD-タスク名.md`

- 実施内容1
- 実施内容2
- 発見した問題（あれば簡潔に）
```

## ポイント

- 詳細は `.claude/logs/` に、概要は `TODO.md` に
- TODO.md から詳細ログへの参照リンクを必ず含める
- 日付は必ず含める（後から検索しやすくするため）
- 変更ファイル一覧は具体的に記載
